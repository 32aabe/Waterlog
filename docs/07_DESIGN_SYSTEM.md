# Design System v0.1

This document translates Waterlog's visual direction into practical UI rules.

It is not final.

It exists to keep the demo visually consistent.

---

# Core Rule

When in doubt, reduce interface.

The user should notice the place before noticing the UI.

---

# Color

Use muted, natural colors.

Prefer:

- warm cream backgrounds
- soft sage green
- fog gray
- wet stone
- deep water blue as accent only

Avoid:

- bright blue as the main color
- pure white surfaces
- harsh black text
- saturated success green
- dashboard-like color coding

---

# Typography

Use calm typography with clear hierarchy.

Display type may feel poetic.

Body text should stay readable and quiet.

Avoid overly technical labels.

Prefer natural language over UI language.

Example:

Use:
"This place now holds this moment."

Avoid:
"Saved successfully."

---

# Layout

Use generous whitespace.

Do not fill every empty area.

Avoid dense dashboards.

Avoid stacked cards unless the content truly needs separation.

Spot pages should feel like pages in a field notebook.

Capture should feel immediate and light.

Map should feel spacious and atmospheric.

---

# Cards and Surfaces

Use cards sparingly.

Prefer soft sections, dividers, and open spacing.

Borders should be subtle.

Shadows should be very light.

No heavy containers unless needed for readability.

---

# Icons

Icons should be simple, thin, and quiet.

Use icons as hints, not decoration.

Avoid icon-heavy interfaces.

Bird icons may be generic.

They do not need to represent exact species.

---

# Illustration

Illustrations may be assembled from reusable parts:

- water type
- water state
- bird activity level
- surrounding natural elements

Bird count is not literal.

It is a visual activity level:

- none
- few
- some
- many

The goal is quick understanding, not scientific precision.

---

# Spot Page

A Spot page has two layers.

## Layer 1: Overview

This is a visual snapshot of the place.

It should show:

- place name
- simple status
- short discovery date
- large visual illustration or photo
- current water state
- bird activity level
- short place character line

It should not feel like a dashboard.

It should not require scrolling.

The user should understand the place at a glance.

## Layer 2: Story

This is the memory layer.

It may be a separate page or view.

It should feel like reading a journal, not browsing a feed.

It may include:

- date
- time
- note
- photo
- observed behavior

For long histories, use filters, calendar, season view, or map view instead of endless scrolling.

---

# Map

Map is a relationship landscape.

Use soft ripple-like markers.

Avoid sharp pins.

Roads and labels should stay visually secondary.

Water and natural areas should feel more important.

---

# Capture

Capture should be fast.

Do not overload it with fields.

AI may fill weather, time, and location quietly.

The user should mainly record:

- what they noticed
- bird or life seen
- water-related behavior
- optional photo

---

# Motion

Motion should feel like water.

Use:

- soft fades
- gentle scale
- slow settling
- ripple-like response

Avoid:

- bounce
- snap
- flashy transitions
- celebration effects

---

# Navigation

Bottom navigation may include:

- Map
- Journal
- Spots
- Profile

Capture remains central and visually primary.

However, navigation should never dominate the screen.

---

# Language

Use English only for the current demo.

Do not mix Korean and English in the interface.

Korean can be added later through localization.
